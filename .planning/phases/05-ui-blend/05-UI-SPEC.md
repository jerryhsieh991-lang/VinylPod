# Phase 5 — UI Design Contract (UI Blend / Professional UX Overhaul)

> Visual and interaction contract for the UI-Blend phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **Scope elevation (user, overriding roadmap default):** this is a *broader UX overhaul* toward a polished, professional macOS-native product — not merely folding mockups into five sizes. The overhaul lives **entirely inside** the frozen liquid-glass token system, the frozen `CONTRACTS.md` data seams, the five window sizes + Dynamic Island, and the six performance invariants. Every ambition below is additive over the existing architecture.

| Property | Value |

**Frozen guardrails this contract must not break:**
- `CONTRACTS.md` data seams — one `NowPlayingService` source of truth, one `updateFromExternal` ingestion path. UI reads the existing view models only; **no new data source, no new `@Published` field, no renamed public API.**
- The six performance invariants (`docs/system-design/05` §3) — most critically **≈0.0% idle CPU** and **≈0.0% CPU when nothing animates**. Steady-playback animation budget is **≤25%** (measured mean ~17.9%) and must fall back to ~0.0% when playback stops.
- Liquid-glass tokens/architecture are **preserved, refined additively** (locked "BLEND UI" decision). Color/motion/radius tokens in frozen `Core/Theme.swift` are **not** rewritten by this phase; refinement happens at the editable `Views/` call sites.
- Five `WindowMode` sizes + Dynamic Island form factor + reused `NSPanel` architecture stay.

## 1. Research Findings

> Multi-source sweep. **Tooling note (honest disclosure):** live web fetch/search was unavailable this session (both Exa and Tavily MCP keys returned 401/Unauthorized; WebSearch/WebFetch returned HTTP 529 Overloaded across repeated retries). HIG findings below are grounded in the cited **primary-source Apple URLs** via the maintained `apple-hig-expert` reference cache (which tracks the live HIG including Liquid Glass, WWDC25 / macOS Tahoe), plus **tool-verified** WCAG contrast + tap-target checks run against VinylPod's *actual* token values, plus the competitive research already captured in `docs/system-design/00-product-vision.md`. Executor should re-verify version-sensitive HIG claims against the live pages at implementation time.

### 1.1 Apple HIG — Liquid Glass & Materials (macOS 26 Tahoe)
- **Liquid Glass** is Apple's translucent material hierarchy (announced WWDC25 Jun 2025; shipped Sep 2025 across iOS/macOS 26). In SwiftUI it is applied via the `glassEffect` modifier; the governing rule is *maintain hierarchy between content and controls — glass is chrome, content stays legible*. VinylPod's `NSVisualEffectView(.hudWindow, .behindWindow)` foundation is the AppKit-native equivalent of "regular" adaptive glass and is retained. Source: https://developer.apple.com/design/human-interface-guidelines/materials · WWDC25 "Meet Liquid Glass" https://developer.apple.com/videos/play/wwdc2025/219/
- **Material weights:** ultra-thin → secondary/floating controls; thin → menus & sidebars; thick → static high-level window backgrounds. Do **not** stack multiple heavy translucent layers (density overload). VinylPod's floating widgets = regular glass; the settings dropdown = light/thin glass; the Reduce-Transparency fallback = an opaque thick-equivalent solid.
- **Vibrancy is a legibility tool, not just transparency** — it pulls background color into text to keep it readable (Vibrant Primary for titles, Vibrant Secondary for captions). Source: HIG Materials (above).
- **Do/Don't:** DO keep text on a material backed by sufficient contrast; DON'T rely on translucency when the backdrop is busy (album art is a busy backdrop) — reduce transparency behind text there. Source: HIG Materials + Accessibility.

### 1.2 Competitor now-playing UIs (from product-vision competitive research)
- Field: Sleeve, Tuneful, NepTunes, Vinyls, Silicio, Jukebox (menu-bar/desktop) and the notch tier (NotchNook, DynamicLake, BoringNotch, Alcove, MediaMate). Source: `docs/system-design/00-product-vision.md` §6.
- **What reads premium** (synthesis): album art is the hero and is razor-sharp; glass/chrome is quiet and recedes; one restrained accent (drawn from the art) not a rainbow; generous consistent padding; type is 2 weights with tight optical spacing; transport is unobtrusive and hover-revealed on larger surfaces; track changes cross-dissolve, they never hard-cut or spin.
- **What reads amateur** (the failure modes to avoid): heavy/bold everywhere, off-grid ad-hoc spacing, low-contrast text floating on bright art, always-on scrolling marquee text, spinners, and accent color sprayed onto every element.
- **VinylPod's differentiators to protect visually:** five real sizes + full-screen desktop mode + album-adaptive glass across *all* surfaces — none of which should be diluted by the overhaul.

### 1.3 macOS menu-bar / NSPanel UX
- Borderless `.nonactivatingPanel`, `becomesKeyOnlyIfNeeded`, transparent background so SwiftUI owns all chrome (existing model, `docs/system-design/02` §1 — keep). Menu-bar apps should not steal key focus; appear/dismiss should be quick and non-jarring.
- macOS is a **precision-pointer** platform (18–30 in viewing distance): the 44×44pt *touch* minimum is the ideal, but the practical pointer minimum hit region is **≥28pt**; glyphs can stay small if the `contentShape` hit region is expanded. Source: HIG Buttons https://developer.apple.com/design/human-interface-guidelines/buttons · HIG Designing-for-macOS https://developer.apple.com/design/human-interface-guidelines/designing-for-macos
- Every primary action needs a keyboard equivalent (already: ⌘1–⌘5, global hotkeys). Source: HIG Designing-for-macOS.

### 1.4 Motion / micro-interactions
- macOS-idiomatic motion is *restrained and physical*: springs for interactive/direct-manipulation feedback, eases for state cross-fades. VinylPod's `VPTheme.spring` (response 0.32, damping 0.7), `VPTheme.fade` (0.45s ease), `VPTheme.liquid` (1.05s ease) already encode this — reuse them, don't invent new curves.
- **When NOT to animate (perf-critical):** never animate on the 1 Hz/10 Hz `position` tick; never run idle/continuous animation; never scroll marquee text at rest. Track-change micro-interactions fire **only** on real `onTrackChanged`. Source: `docs/system-design/05` §3 Rules 1–6.
- SF Symbol state changes (play↔pause) should use the native `.contentTransition(.symbolEffect(.replace))` morph (macOS 14+), which is event-driven, not continuous.

### 1.5 Accessibility (tool-verified against VinylPod tokens)
- **Contrast minimums:** normal text 4.5:1, large text 3:1 (HIG mirrors WCAG 2.2). Source: HIG Accessibility https://developer.apple.com/design/human-interface-guidelines/accessibility · WCAG https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html
- **Tool-verified (the smoking gun for UIB-03):** white title text over a **bright album region with no scrim = 1.56:1 (FAIL)**; over a mid region = 3.45:1 (FAIL); over the existing dark legibility scrim (≈black@0.42 composited) = **11.37:1 (PASS)**. ⇒ the legibility scrim/vignette is **load-bearing, not decorative** — text legibility must never depend on the album hue.
- **Tool-verified tap targets:** Small-mode transport glyph frames (~17–24pt) **FAIL** the 44pt guideline; expand hit regions (see §4/§6) — this is a real finding to fix.
- **Reduce Transparency**: HIG says drop translucency behind text when busy — VinylPod needs an explicit opaque **solid-glass fallback** (currently **absent** — `grep` found zero `accessibilityReduceTransparency` usage). Source: HIG Materials/Accessibility.
- **Reduce Motion / VoiceOver**: currently near-absent — only **2** `.accessibilityLabel` calls exist in the whole app (`grep`-verified); transport controls have **no** VoiceOver labels. Both are required deliverables of this overhaul.

## 2. Spacing Scale

Declared values (strict 4/8pt grid — HIG "8pt rule"):

| Token | Value | Usage |
|-------|-------|-------|
| hairline | 1px | Strokes/borders only (`glassStroke`, rim) |
| 2xs | 2px | Optical nudges only (never layout) |
| xs | 4px | Icon-to-label gaps, inline padding |
| sm | 8px | Compact element spacing, chip padding |
| md− | 12px | Title↔artist gap, tight control rows |
| md | 16px | Default element spacing, panel inset |
| lg | 24px | Section padding, art↔text gap (larger sizes) |
| xl | 32px | Layout gaps, desktop chrome insets |
| 2xl | 48px | Major section breaks (desktop) |
| 3xl | 64px | Full-screen composition margins |

**Exceptions (documented):**
- **Hit-target floor 28pt** (macOS pointer minimum) via expanded `contentShape`; **44pt** is the ideal where footprint allows. Off-grid because it is an ergonomic floor, not a layout rhythm.
- Existing per-size widget container radii (`18`/`12`) and small art insets (`7`) are retained where already tuned; new/refined spacing snaps to the grid above.
- Desktop numeric timer optical alignment may use sub-grid nudges (≤2px).

## 3. Typography

**System: SF Pro. Exactly 2 weights — Regular (400) + Semibold (600).** This is the core professionalizing move: replace the current sprawl of `.heavy`/`.bold`/`.semibold`/`.medium` at inline `Views/` call sites with two weights. Rationale: HIG restraint + macOS-native feel; Heavy/Bold at small sizes reads amateur/heavy-handed. Sizes map to macOS system text metrics.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | Semibold (600) | 1.1 | Desktop caption title; **exception:** desktop full-screen timer scales to 96–138px, `.monospacedDigit()`, still weight 600 |
| Title | 17px | Semibold (600) | 1.15 | Track title (Medium/Large/Island expanded); Small drops to 13px Semibold for footprint |
| Body | 13px | Regular (400) | 1.3 | Artist, secondary metadata, empty-state body |
| Caption | 11px | Regular (400) | 1.2 | Time labels, source badge, status, hints |

**Notes / deviations (with rationale):**
- Web-default 1.5 body leading is **deliberately rejected** — too airy for dense macOS widgets. Titles are single-line (`lineLimit(1)`, tail truncation); body leading 1.3.
- Dynamic Island retains `design: .rounded` for title/artist (friendlier register, existing decision) — same 2-weight rule applies.
- **FROZEN-token FLAG (resolve in planning):** `VPTheme.title()/caption()` helpers emit `.bold`. They live in frozen `Core/Theme.swift` and are mostly used by menus/status, not widgets. This phase converges the editable *inline* `Views/` call sites to Regular/Semibold and treats those helpers' `.bold` as "emphasis" without editing Core. Softening the helpers to `.semibold` is an **optional, low-risk** refinement deferred to the Phase 3 `VinylPodKit` refactor when Core is legitimately touched — **not** performed by this phase. Do not edit `Core/Theme.swift` here.
- **Dynamic Type:** widgets are fixed-size chrome, but text must not clip; support at least one step of enlargement gracefully (truncate, never overflow).

## 4. Color (60 / 30 / 10)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Static landscape — deep midnight-ice base (`#142E4D`→`#0E1A2E`) under a fixed dark scrim | Background behind every mode; **never** track-reactive (hard rule) |
| Secondary (30%) | Album-reactive glass — `VisualEffectBlur(.hudWindow,.behindWindow)` + `dominant`/`muted` membrane | Floating widget panels, Dynamic Island, settings dropdown (light variant) |
| Accent (10%) | Album `vibrant` token (== `settings.accentColor`); fallback `VPTheme.iceAccent` `#61C7F5` | see reserved list below |
| Destructive | `Color(nsColor: .systemRed)` | Quit / Last.fm-disconnect confirmations only (not surfaced in widget chrome this phase) |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. Progress-bar fill + knob
2. Play/Pause button fill + its glow shadow
3. Active source-provenance badge (when it lands, Phase 4)
4. Drag-and-drop target highlight + border
5. Keyboard focus ring

**Accent must NOT touch:** body/title text, default icon color (icons are `textPrimary`/`textSecondary` white), panel backgrounds, borders (except the focus ring), the landscape.

**60/30/10 discipline:** the landscape dominates, glass is the quiet secondary surface, and exactly one album-drawn accent carries interaction. This is the anti-"glass everywhere maximalism" rule from REQUIREMENTS Out-of-Scope — *legibility wins over translucency, always.*

## 6. Material & Radius Contract

### Materials (HIG mapping)
| Surface | Material | Reduce-Transparency fallback |
|---------|----------|------------------------------|
| Floating widgets (Small/Medium/Regular/Large) | `NSVisualEffectView(.hudWindow, .behindWindow)` = "regular" glass | Opaque solid: `shadow`-token @ ~0.96 (or `.windowBackgroundColor`), blur removed, album membrane flattened to `dominant`@0.18 tint on solid; art kept |
| Dynamic Island (capsule + panel) | `.hudWindow` bespoke stacks (`IslandCapsuleGlass`/`IslandPanelGlass`) | Same opaque-solid rule; capsule becomes solid dark pill |
| Settings dropdown | `.hudWindow` light glass (`white@0.93` + `vibrant@0.11`) | Opaque `white@0.98`; ink to full `black@0.92`; border opacity +0.1 under Increase Contrast |
| Desktop canvas | Bespoke ambience recipe (`DesktopWidgetCanvas`, higher-opacity gradients) | Reduce blur contribution to 0; rely on gradient wash + fixed scrim |

### Radius scale (preserve frozen `VPTheme` radii; apply concentricity)
| Token | Value | Used for |
|-------|-------|----------|
| `radiusSmall` | 10pt | Chips, small controls, inner art on Small |
| `radius` | 14pt | Standard album-art tiles, medium controls |
| `radiusLarge` | 22pt | Window shells, large overlay panels |
| per-size container | 18pt (Small/Medium), 12pt (Regular/Large) | existing, retained |

**Concentricity rule (HIG):** nested corners must be concentric — inner art radius = container radius − art inset (e.g., container 18 with 8pt inset → art radius 10). Always `style: .continuous`. This is a professionalizing refinement over the current ad-hoc art radii (5/7).

## 7. Component Contracts

Each below: refined layout on the 8pt grid, 2-weight type, accent discipline, hit-region + VoiceOver. All reads come from existing view models only.

### 7.1 Album Art (hero element, all sizes)
- Razor-sharp, `scaledToFill`, concentric-radius clip, 1px `glassStroke`, soft drop shadow (`black@0.30`, r12, y6).
- Loading → static `music.note` placeholder on `panel`; **cross-dissolve** to art on real track change (`VPTheme.fade`, trigger = `nowPlaying.track` identity).
- VoiceOver: label = "Album artwork" + value "{title} by {artist}"; `.accessibilityHidden(true)` if a redundant adjacent label already reads it.

### 7.2 Transport Controls (`TransportControls.swift`)
- Prev / Play-Pause / Next. Play/Pause is the only accent-filled element; skip glyphs are `textPrimary` white.
- Play/Pause glyph uses `.contentTransition(.symbolEffect(.replace))` (event-driven morph on `isPlaying`).
- Hover scale 1.08 via `VPTheme.spring` (input-triggered).
- **Hit region ≥28pt (aim 44pt) via `contentShape(Rectangle())`** — fixes the tool-verified 17–24pt failure; glyph size unchanged.
- VoiceOver labels (required): "Previous track", "Play"/"Pause" (state-dependent), "Next track". Add `.accessibilityHint` where useful.

### 7.3 Progress Bar (`ProgressBarView.swift`)
- Track `panel`; fill + knob `accentColor`; time labels `caption`/`textMuted` `.monospacedDigit()`.
- **The fill width has NO `.animation` on `fraction`** — it tracks position and must update at ≤1 Hz via `Int(position)` coarsening (Rule 3). Never animate width per tick.
- Knob scales 1.25 on drag (`VPTheme.spring`, input-triggered).
- VoiceOver: `.accessibilityValue("{elapsed} of {duration}")`; drag = adjustable action.

### 7.4 Small (162×162)
- Art top-left (98pt, radius 10 concentric), bottom 42pt glass strip: title 13pt Semibold + prev/play/next. Grid: 8pt strip padding, 4pt icon gaps.
- Close (top-left, always visible) + settings ellipsis (top-right). Expand both hit regions to ≥28pt.

### 7.5 Medium (344×132)
- Art left 100pt (radius 12), text column: title 17pt Semibold + artist 13pt Regular (12pt gap), transport + optional progress strip. Column inset 16pt.
- Progress strip stays an isolated leaf reading coarsened `position` (perf pattern).

### 7.6 Regular (300×360)
- Full-bleed art; bottom caption gradient (title 17pt Semibold / artist 13pt Regular). Transport row hover-revealed (`VPTheme.fade`).
- Caption darkens more over bright covers (`isBrightArtwork`) — keep; it is the legibility guarantee.

### 7.7 Large (320×432)
- Centered art card 260pt (radius 14, or `VinylDiskView` in Vinyl style), title 17pt Semibold / artist 13pt Regular below, transport + optional progress. Close+settings hover-gated to 0.42→1.0 opacity.

### 7.8 Desktop (full screen)
- Vinyl deck (tilted cover + spinning record + tonearm), big monospaced timer (Display role, 96–138pt, weight 600), playback block bottom-left (title 28pt / artist 18pt→map to Display/Body). Chrome hover-revealed. Distinct ambience recipe retained.
- Vinyl spin + timer are the **known deliberate playback-cost animations** (see §9).

### 7.9 Dynamic Island (capsule + expanded)
- Compact: 22pt thumb, "Artist • Title", equalizer bars, chevron, settings. Expanded: large art, title 17–28pt Semibold `.rounded`, `IslandTimeRow` (sole `position` reader, coarsened), transport, equalizer.
- Keep aggressive leaf-splitting (island root does NOT observe `NowPlayingService`).

### 7.10 Settings dropdown / chrome buttons
- Light glass; ink `black@0.72–0.92`; **no white text on this surface** (existing rule). Increase Contrast → ink to full black, borders +0.1.
- All chrome buttons (close, settings ellipsis, layer toggle) get VoiceOver labels + ≥28pt hit regions.

## 8. Interaction & Motion Contracts

Every animated element declares its **trigger** and confirms it is **not** the position tick. (Full proof matrix in §9.)

| # | Interaction | Curve | Trigger (must be real-change/input) | Reduce-Motion behavior |
|---|-------------|-------|-------------------------------------|------------------------|
| M1 | Album-art cross-dissolve | `VPTheme.fade` 0.45s | `nowPlaying.track` identity change (`onTrackChanged`) | Hard cut (instant) |
| M2 | Glass/landscape palette cross-fade | `VPTheme.liquid` 1.05s | `settings.albumPalette` change (dedup-gated, real palette only) | Instant palette swap (no cross-fade) |
| M3 | Play↔Pause glyph morph | `.symbolEffect(.replace)` / `VPTheme.spring` | `nowPlaying.isPlaying` change | Instant glyph swap |
| M4 | Title/artist text change | `VPTheme.fade` 0.45s | `nowPlaying.track` change | Instant |
| M5 | Mode/size cross-fade | `VPTheme.fade` 0.45s opacity | `mode` change (user action) — no `.id(mode)` pinning (Rule 5) | Instant (opacity 0→1 ≤0.1s) |
| M6 | Transport/knob hover & press | `VPTheme.spring` | hover / press (user input) | Disable scale; keep state change |
| M7 | Chrome hover-reveal (Regular/Large/Desktop) | `VPTheme.fade` | `.onHover` (user input) | Show controls at rest (no fade) |
| M8 | Drop-target highlight | `VPTheme.fade` | `isDropTargeted` (user drag) | Instant border |
| M9 | Island expand/collapse | spring 0.44/0.86 | user tap on chevron | Instant size change |
| M10 | Long-title handling | **static truncation (tail)**; optional marquee **only on hover** | hover (never at rest) | Marquee disabled entirely; truncate |
| M11 | Equalizer bars | `TimelineView` ≤30fps | `isPlaying == true` only; **paused when stopped** | Freeze to static bar glyph |
| M12 | Desktop vinyl spin + timer | linear 18s/rev; timer 10 Hz | `isPlaying == true` only | Static record; timer updates without spin |

**Hard motion rules (restating perf invariants):**
- **No animation fires on the 1 Hz/10 Hz `position` tick.** M1–M10 are all real-change/input triggered. M11–M12 are the *deliberate playback-cost* animations, gated on `isPlaying` and paused at stop.
- **No new idle/continuous animation may be introduced.** No always-on marquee (M10), no resting pulse/shimmer, no spinner.
- Reuse `VPTheme.fade/liquid/spring` — do not invent new curves.

## 9. Performance-Invariant Compliance (mandatory)

Maps **every** animated/dynamic element to its trigger, proving ≈0.0% idle CPU and ≈0.0% when nothing animates. Enforced by the Phase 1 perf-guard tests and re-profiled per UIB-04.

| Element | Fires on | 1Hz/10Hz position tick? | Idle when no track / stopped? | Notes |
|---------|----------|------------------------|-------------------------------|-------|
| Album-art cross-dissolve (M1) | `onTrackChanged` | **No** | Yes (0%) | Track identity is change-gated in `updateFromExternal` |
| Palette cross-fade (M2) | `albumPalette` change | **No** | Yes (0%) | `setAlbumPalette` dedup guard (`palette != albumPalette`) blocks re-fire |
| Play/Pause morph (M3) | `isPlaying` change | **No** | Yes (0%) | State edge only |
| Text change (M4) | `track` change | **No** | Yes (0%) | Coarsened; equality-gated |
| Mode cross-fade (M5) | user size switch | **No** | Yes (0%) | `.transition(.opacity)`, stable `.id("vinylpod.content")`, `modeTransitionInFlight` guard |
| Hover/press (M6, M7, M8) | user input | **No** | Yes (0%) | No input = no work |
| Progress fill (7.3) | coarsened `Int(position)` ≤1Hz | **No** (coarsened, not per-raw-tick; no width animation) | Yes (0%) | Leaf-isolated; Rule 3 |
| Island root body | — | **No** (does not observe `NowPlayingService`) | Yes (0%) | Leaf-split; `IslandTimeRow` sole `position` reader |
| Equalizer bars (M11) | `isPlaying==true` | No (own ≤30fps timeline) | **Yes — paused at stop** | Counts toward ≤25% playback budget only |
| Desktop vinyl/timer (M12) | `isPlaying==true` | No | **Yes — stops at pause** | Counts toward ≤25% playback budget only |

**Gate:** any always-on parent view must NOT observe `NowPlayingService` (Rule 1); `position` stays the only unconditionally-written field (Rule 2); leaf views coarsen to whole seconds (Rule 3); `setAlbumPalette` only on real track change (Rule 4); size-switch uses opacity cross-fade not `.id(mode)` (Rule 5); `modeTransitionInFlight` guard stays (Rule 6). **`sample`/`powermetrics` must confirm ~0.0% idle after each UI batch before phase sign-off (UIB-04).**
